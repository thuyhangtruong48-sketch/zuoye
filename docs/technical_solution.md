# 避开危险路段的救援路径规划技术方案

## 1. 项目场景

灾害发生后，救援车辆从出发点前往受灾点时，不能只追求距离最短。道路可能受到塌方、积水、拥堵或封控影响，最短路线可能反而不适合救援通行。本项目将道路抽象为图结构，将灾害风险转化为道路边权重，使用 Dijkstra 算法规划普通最短路径和安全路径。

## 2. 数据来源

本项目使用两类真实数据：

- 真实地图道路数据：通过高德 Web 服务路径规划接口获取。字段包括道路名称、导航指令、道路距离、路线坐标和交通状态。
- 真实历史灾害资料：记录在 `data/historical_disasters/disaster_events.csv`。地震场景采用 1679 年三河-平谷地震公开历史资料；洪水场景采用 2012 年北京“7·21”特大暴雨及中国气象数据服务资料。

需要说明的是，公开历史灾害数据通常不会直接给出“某条道路是否封闭或损坏”的逐路段记录。因此本项目采用空间叠加方法：用真实灾害事件确定影响区，再把影响区映射到真实道路网络上。

## 3. 道路图模型

道路网络被抽象为无向图：

- 节点：高德路线坐标点、起点、终点和关键转向点。
- 边：相邻节点之间的道路段。
- 边属性：距离、道路名称、拥堵程度、通行状态、危险类型和综合权重。

核心数据文件包括：

- `data/amap/nodes.csv`：高德路线节点。
- `data/amap/edges.csv`：高德路线道路边。
- `data/amap_earthquake/edges.csv`：地震场景道路边。
- `data/amap_flood/edges.csv`：洪水场景道路边。
- `road_disaster_mapping.csv`：道路与历史灾害影响区的叠加结果。

## 4. 历史灾害映射方法

本项目不再仅按道路名称指定某条路为危险路段，而是采用如下流程：

1. 记录历史灾害事件，包括灾害类型、时间、位置、影响半径、来源和说明。
2. 根据灾害点或代表性影响位置建立缓冲区。
3. 将高德返回的道路边视为线段。
4. 计算道路边到灾害影响区中心的最短距离。
5. 如果道路边与灾害影响区相交，则标记为塌方或积水风险。

地震场景中，进入历史地震影响缓冲区的道路被标记为 `collapse`。洪水场景中，进入历史洪水影响缓冲区的道路被标记为 `flood`。没有进入灾害区但拥堵程度较高的道路标记为 `congestion`。

## 5. 权重设计

普通最短路径只考虑距离：

```text
distance_weight = distance
```

安全路径综合考虑距离、灾害风险和拥堵：

```text
safe_weight = distance * risk_factor * (1 + congestion_weight * congestion)
```

风险系数设置如下：

| 危险类型 | 含义 | 风险系数 |
| --- | --- | ---: |
| normal | 正常通行 | 1.0 |
| congestion | 拥堵 | 1.4 |
| flood | 积水风险 | 2.2 |
| collapse | 塌方风险 | 4.0 |

## 6. Dijkstra 算法过程

Dijkstra 算法适用于非负权重图。本项目所有道路距离和安全权重均为非负数，因此适合使用该算法。

算法步骤：

1. 将起点代价设为 0，其余节点代价设为无穷大。
2. 每次从优先队列中取出当前总代价最小的节点。
3. 遍历该节点相邻道路，计算经过该道路后的新代价。
4. 如果新代价更小，则更新节点代价和前驱节点。
5. 到达终点后，根据前驱节点回溯得到完整路径。

本项目运行两次 Dijkstra：

- `distance` 模式：边权重为道路距离，得到普通最短路径。
- `safe` 模式：边权重为综合安全权重，得到安全路径。

## 7. 当前实验结果

| 场景 | 路径 | 总距离 | 危险类型 | 结果解释 |
| --- | --- | ---: | --- | --- |
| 地震 | 普通最短路径 | 19.973 km | collapse, congestion | 距离较短，但经过历史地震影响区 |
| 地震 | 安全路径 | 26.816 km | congestion | 绕开主要塌方风险，但距离增加 |
| 洪水 | 普通最短路径 | 19.973 km | congestion, flood | 距离较短，但经过历史洪水影响区 |
| 洪水 | 安全路径 | 26.816 km | congestion | 绕开主要积水风险，但距离增加 |

结果说明：在灾害救援场景中，普通最短路径不一定是最适合救援的路线。安全路径通过提高灾害影响区内道路的权重，引导算法选择绕行路线，从而降低救援车辆进入高风险区域的可能性。

## 9. 改进方向
